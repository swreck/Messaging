## What this milestone is

The Tier 1 Resolution Loop.

The methodology rule Ken articulated in this session: a Tier 1 value statement must RESOLVE the specific question inside the audience's priority ‚Äî not echo its topic. This version of Maria catches when she can't resolve a priority, interviews the user for the missing differentiator, and regenerates the Tier 1.

## What's in it

### Core: the Tier 1 Resolution Loop

1. **Gate 1 ‚Äî Mapping (`backend/src/prompts/mapping.ts`).** The match test is MF-against-Driver. The prompt names three specific failure modes, each with a worked counterexample:
   - **PATTERN-FROM-HISTORY** ‚Äî "X has happened before, so X will happen again" (Sofia's tenure ‚Üí donor loss).
   - **CHAIN-OF-REASONING** ‚Äî MF requires the reader to supply inference steps to reach the question (SQL-compat ‚Üí "will migration blow up weekends?").
   - **AUDIENCE-SCOPE MISMATCH** ‚Äî MF satisfies a connected party but not the one named in the Driver (510(k) clearance satisfies CMO, not the cardiology team).
   When no honest match exists, mapping emits a `gapDescription` with a one-sentence description of what kind of differentiator would close the gap.

2. **Gate 2 ‚Äî Maria interview (`backend/src/prompts/partner.ts` HANDLING MAPPING GAPS section).** When mapping flags a gap, Maria asks the user directly (*"We know they care most about X ‚Äî nothing in the offering I have answers that. What am I missing?"*), extracts the new differentiator via `add_capabilities`, drafts its MF via `edit_capabilities`, then triggers rebuild.

3. **Rebuild ‚Äî endpoint + UI button + Maria action, UI/Maria parity.**
   - `POST /api/express/rebuild-foundation` ‚Üí `rebuildFoundationFromDraft` in `backend/src/lib/expressPipeline.ts`. Wipes mappings + tier rows, re-runs mapping + tier generation against current DB state, returns new `FoundationResult` including any remaining gaps.
   - "Rebuild Tier 1 with updated info" button in `frontend/src/guided/GuidedFlow.tsx` beneath the foundation card.
   - `rebuild_foundation` Maria action in `backend/src/lib/actions.ts`. Response is a `[FOUNDATION_REBUILT]<json>` marker the frontend parses to update the foundation-card in place.
